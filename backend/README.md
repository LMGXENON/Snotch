# Snotch Backend

Production-oriented starter backend for licensing, validation, abuse protection, and script generation.

## What this includes
- `POST /v1/license/activate`
- `POST /v1/license/validate`
- `POST /v1/generate/script`
- `POST /v1/generate/bundles`
- rate limiting on sensitive endpoints
- local JSON license store for starter setup (`data/licenses.json`)

## Quick start
1. Install dependencies:
   - `cd backend`
   - `npm install`
2. Configure environment:
   - `cp .env.example .env`
   - run `npm run secrets` and copy values into `.env`
   - set `OPENAI_API_KEY`, `JWT_SECRET`, `LICENSE_PEPPER`, `ADMIN_API_KEY`
3. Import license hashes from CSV generated by app script:
   - `npm run import:licenses -- ../licenses.csv`
4. Run backend:
   - `npm run dev`

## Domain + hosting
- Deploy to Railway/Render/Fly/Cloudflare Workers-equivalent Node hosting.
- Point `api.yourdomain.com` to deployed backend.
- Enable HTTPS.
- Update app URLs in `Snotch/LicenseManager.swift`.

## Local development URL
- Backend runs at `http://localhost:8787` by default.
- App uses localhost automatically in Debug builds.
- App uses `https://api.snotch.app` in Release builds.

## Generate keys
Use script in app repo root:
- `python3 scripts/generate_license_keys.py --count 100 --prefix SNTCH --out licenses.csv --pepper "YOUR_SERVER_SECRET"`

## Production notes
- Store only `license_hash` in database.
- Move from `data/licenses.json` to Postgres for production.
- Add IP reputation + geo anomaly rules for abuse control.
- Keep OpenAI key server-side only.

## Admin access
- Send header `x-admin-key: <ADMIN_API_KEY>`.
- With `ADMIN_BYPASS_UNLIMITED=true`, admin bypasses all rate limits.
- Admin can call generation endpoints unlimited times.
- Admin can call `GET /v1/admin/licenses` to inspect licenses.
